Why Supabase

Auth — free

JWT, refresh tokens, email verification, password reset, OAuth — all handled. Zero custom auth code.

API — auto-generated

PostgREST turns every table into a full REST API instantly. CRUD endpoints need no server code.

Realtime — built-in

PostgreSQL replication → WebSocket push. Phase 3 real-time sync is essentially free.

Storage — included

S3-compatible file storage for Phase 2 note attachments. No extra infra.

Background jobs — Trigger.dev

Open-source, TypeScript-native, self-hostable. Handles reminders, recurring tasks, emails — with delay, retry, and observability built in.

Local dev — Supabase CLI

One command spins up a full local Supabase stack (Postgres, Auth, Storage, Edge Functions) via Docker.

PostgREST handles ~80% of the API surface automatically. Edge Functions only exist for logic that PostgREST can't express: aggregated dashboard data, global full-text search across multiple tables, file export, and incoming job webhooks.

Background Jobs — Trigger.dev v3

Why Trigger.dev v3 over alternatives: It is open-source (MIT), TypeScript-native, self-hostable on your own infra later, has a native Supabase integration (DB webhook → job trigger), and supports wait.for delays — meaning a reminder can literally pause inside the job until the exact moment it should fire, without a polling loop. Inngest is the closest alternative but is closed-source. pg_cron is fine for simple DB maintenance but too limited for retries, fan-out, and complex email logic.

Jobs — Phase 1

reminder.schedule

Triggered when a reminder row is inserted. Uses wait.until(remind_at) then sends email/push via Resend. Cancellable by job ID.

email.verification

Triggered on new user signup (Supabase Auth webhook). Sends branded verification email.

email.passwordReset

Triggered by Supabase Auth password-reset hook. Sends custom-styled reset email.

Jobs — Phase 2

task.recur

Cron job that runs daily. Queries tasks with recurrence IS NOT NULL and completed, generates the next occurrence, inserts new task row.

digest.daily

Optional opt-in daily digest email: today's tasks + overdue summary. Scheduled per user timezone.

export.generate

Long-running job triggered by user export request. Builds ZIP of Markdown notes + JSON tasks, uploads to Supabase Storage, sends download link.

Example job — reminder

// jobs/reminder.ts import { task, wait } from "@trigger.dev/sdk/v3"; import { resend } from "../lib/email"; export const reminderJob = task({ id: "reminder.schedule", run: async (payload: { reminderId: string; userId: string; remindAt: string; message: string; channel: string }) => { // pauses execution — no polling, no cron tick overhead await wait.until({ date: new Date(payload.remindAt) }); if (payload.channel === "email") { await resend.emails.send({ ... }); } // mark reminder as sent in DB await supabase.from("reminders").update({ is_sent: true }).eq("id", payload.reminderId); }, });

Row Level Security (RLS)

RLS replaces all custom auth middleware. Every table is locked to the authenticated user via auth.uid(). No user can ever read or modify another user's data — enforced at the database level.

-- notes: users can only see and modify their own rows ALTER TABLE notes ENABLE ROW LEVEL SECURITY; CREATE POLICY "notes: owner full access" ON notes FOR ALL USING (user_id = auth.uid()) WITH CHECK (user_id = auth.uid()); -- tasks: access via the parent list's ownership CREATE POLICY "tasks: owner via list" ON tasks FOR ALL USING ( list_id IN ( SELECT id FROM todo_lists WHERE user_id = auth.uid() ) );
Tip: The Supabase client sends the user's JWT automatically. PostgREST reads it, sets auth.uid(), and RLS does the rest. No auth headers to validate manually in application code.

Build Milestones

1

Supabase project + schema

Create Supabase project, run migrations (tables + indexes + tsvector), write all RLS policies, create handle_new_user DB trigger to auto-insert into profiles. Verify locally with Supabase CLI.

2

Auth (zero custom code)

Enable email auth + Google/GitHub OAuth in Supabase dashboard. Configure email templates. Test full sign-up → verify → login → refresh → logout flow via supabase-js. Set up Auth webhook to call email.verification job in Trigger.dev.

3

Notes + Tags API

PostgREST handles all CRUD automatically after RLS policies are in place. Test filtering, pagination, tag attach/detach. Verify RLS isolation between two test users.

4

To-Do Lists + Tasks API

PostgREST CRUD for lists and tasks. Test sub-tasks (parent_id), priority filtering, completion toggle, and cascade deletes. Verify drag-order updates via bulk PATCH.

5

Edge Functions: Dashboard + Search

Deploy dashboard and search Edge Functions. Dashboard aggregates pinned notes, today's tasks, overdue tasks in one DB call. Search uses to_tsquery on notes + ilike on tasks.

6

Trigger.dev + Reminders

Set up Trigger.dev v3 project. Deploy reminder.ts job. Wire Supabase DB Webhook on reminders INSERT → webhooks/trigger Edge Function → Trigger.dev. Deploy reminder-cancel Edge Function. Test full flow: create reminder → job fires at correct time → email sent via Resend → is_sent updated.

7

Phase 2: Storage, Recurring Tasks, Export, Digest

Supabase Storage for note attachments. Trigger.dev recurring-tasks.ts cron job. export.ts job (builds ZIP → uploads to Storage → emails link). Optional daily digest cron.

8

Phase 3: Realtime sync + Shared workspaces

Enable Supabase Realtime on notes and tasks tables. Frontend subscribes via supabase-js. Add workspace_id to schema and RLS policies for team sharing.
